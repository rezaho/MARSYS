# Framework features required for Spren v0.3

The single framework feature blocking Spren v0.3 is the **NDJSON streaming tracing writer**. Everything else Spren v0.3 needs is already in the framework after the unified-barrier merge.

## Outcome

When Spren v0.3 ships:

- The framework's tracing layer writes events to disk **as they are emitted**, line-by-line, in NDJSON format.
- Mid-run process crashes do NOT lose the partial trace; readers can tail-follow the file.
- The current write-at-end JSON writer is replaced. Existing framework consumers are unaffected because the writer interface is preserved; the persistence format moves from JSON-at-completion to NDJSON-streaming.
- Framework's own regression suite passes with zero new failures.

EventBus subscriber appends one JSON object per line per event. Reader interface returns a hierarchical TraceTree on demand. Framework regression suite green. |

## Why only this one for v0.3

Spren v0.3 consumes the framework as it stands today (post-unified-barrier merge) plus the NDJSON change. Everything else Spren v0.3 needs is Spren-side work:

- Workflow CRUD + types — built in Spren's Pydantic layer
- Python-file workflow import — Spren-side AST parser; no framework change needed
- Run execution + SSE event stream — Spren subscribes to the framework's existing `EventBus`; the AG-UI translator is Spren-side
- Cost rate table + per-run aggregation — Spren-side; the framework already emits token usage on `EventBus`
- Memory subsystem — entirely Spren-side
- Meta-agent core + capabilities — entirely Spren-side
- Home page UI — Spren-side
- Native distribution + Tauri shell + native installer — Spren-side packaging

If Session 01 surfaces a hidden dependency on a TRUNK-CRITICAL change, it gets escalated for ADR review before any work begins on that secondary change.

## Acceptance for Spren v0.3 from the framework's side

- [ ] NDJSON streaming tracing writer merged to framework main
- [ ] Framework release that includes it tagged + published (the framework team picks the version label — could be a v0.3.x patch or a minor)
- [ ] Spren's Session 04 brief references the released framework version
- [ ] Framework regression suite green at the released SHA

## Risks

| Risk | Mitigation |
|---|---|
| **Streaming writer surfaces an existing race condition in the EventBus** | Brief includes explicit ordering tests; if races appear, fix or escalate; do NOT paper over with a buffer |
| **Framework users relying on the legacy JSON-at-end format** | Document migration path in framework's CHANGELOG. Old `.json` trace files remain readable via a small util (one-shot conversion); post-change writer only emits NDJSON. SP-006 equivalent — no dual-write |
| **Per-line `fsync` cost** | Default to no `fsync` per line; trust OS buffers. Provide an opt-in `--fsync-events` flag for users who need stronger durability. Benchmark before shipping |
| **Existing framework tests assume the JSON-at-end shape** | All such tests get migrated as part of Session 01; framework regression suite must remain green |
