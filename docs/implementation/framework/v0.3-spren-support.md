# Framework features required for Spren v0.3

Two framework features block Spren v0.3: the **NDJSON streaming tracing writer** (Session 01, ✅ shipped) and the **AG-UI event stream translator** (Session 06, ✅ shipped). Everything else Spren v0.3 needs is already in the framework after the unified-barrier merge.

## Outcome

When Spren v0.3 ships:

- The framework's tracing layer writes events to disk **as they are emitted**, line-by-line, in NDJSON format.
- Mid-run process crashes do NOT lose the partial trace; readers can tail-follow the file.
- The framework ships an `AGUIEventStream(orchestra, run_id) -> AsyncIterator[AGUIEvent]` adapter that translates `EventBus` events into AG-UI-format events for any consumer (Spren, MARSYS Cloud, third-party hosted dashboards, custom UIs).
- The current write-at-end JSON writer is replaced. Existing framework consumers are unaffected because the writer interface is preserved; the persistence format moves from JSON-at-completion to NDJSON-streaming.
- Framework's own regression suite passes with zero new failures.

## Why these two for v0.3

Spren v0.3 consumes the framework as it stands today (post-unified-barrier merge + Session 01 shipped) plus the AG-UI translator. Everything else Spren v0.3 needs is Spren-side work:

- Workflow CRUD + types — built in Spren's Pydantic layer
- Python-file workflow import — Spren-side AST parser; no framework change needed
- Run execution + SSE event stream — Spren wraps the framework's `AGUIEventStream` adapter in an SSE HTTP endpoint; the translation logic itself is framework-side (Session 06)
- Cost rate table + per-run aggregation — Spren-side; the framework already emits token usage on `EventBus`
- Memory subsystem — entirely Spren-side
- Meta-agent core + capabilities — entirely Spren-side
- Home page UI — Spren-side
- Native distribution + Tauri shell + native installer — Spren-side packaging

If Session 06 surfaces a hidden dependency on a TRUNK-CRITICAL change, it gets escalated for ADR review before any work begins on that secondary change.

## Acceptance for Spren v0.3 from the framework's side

- [x] NDJSON streaming tracing writer (Session 01) merged + released
- [ ] AG-UI event stream translator (Session 06) merged to framework main

## Risks

| Risk | Mitigation |
|---|---|
| **AG-UI translator delayed past Spren Session 04 start** | Spren Session 04 can develop against a draft branch of the framework PR (vendor via uv branch reference) and switch to the published version on merge. Forbidden: forking a "temporary" Spren-side translator — that becomes permanent debt. |
| **AG-UI protocol still evolving** | Pin to a specific AG-UI version in Session 06's brief; use AG-UI's `Custom` event for marsys-specific events that don't fit; ship `schema_version: int` on every event so consumers can detect protocol drift. |
| **EventBus shape mismatches AG-UI event granularity** | Session 06's brief includes a complete event-to-event mapping table written up-front; if mappings reveal a missing framework event type, the brief escalates before code is written. |
