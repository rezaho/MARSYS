## 1. Goal

A workflow that uses a specialized `Agent` subclass (`WebSearchAgent`, `BrowserAgent`, `CodeExecutionAgent`, `DataAnalysisAgent`, `FileOperationAgent`) must serialize and reload **as that subclass**, with its specialized tools/instructions/methods intact — not flatten to base `Agent`. The Spren consumer authors it the way a Python file does (pick the class, pass its declarative config); the class rebuilds its own behaviour on load. The base `Agent` round-trip stays byte-identical (no migration of existing specs).

**The gap, confirmed in code:** `agent_to_pydantic` passes the `isinstance(agent, Agent)` guard for all 5 subclasses (`agents/serialize.py:85`) but extracts only base fields (`:110-125`); `pydantic_to_agents` unconditionally rebuilds a base `Agent` (`:158`). All subclass-specific config is silently dropped. `agents/serialize.py` is the only agent serializer (no subclass dispatch anywhere). ADR-008 Decision 9 / Alternative 6 (`ADR-008:75,96,104`) blessed closing this in Session 09 with the discriminator + closed registry + behaviour-in-class pattern.

## 2. The contract (decided with the user — re-confirm against code in Phase B, do not trust this prose)

- **Discriminator + params, additive.** `AgentSpec` gains `kind: str = "agent"` (default → base `Agent`, existing specs byte-identical, no migration) and `params: <typed>` carrying the subclass's declarative config. `kind`/`params` are **additive on top of the existing flat `AgentSpec` surface** — every base field (`name/goal/instruction/model/tools/allowed_peers/is_convergence_point/memory_*/plan_config/schemas`) still round-trips for a subclass. `kind` mirrors `NodeSpec.kind` (`topology/serialize.py:131`) — one discriminator convention across the wire. The word "type" is not used; no `agent_` prefix on the new fields.
- **`agent_model` → `model` rename, bundled here.** Verified empirically (pydantic 2.13.3): a field named exactly `model` is clean — zero warnings, no `ConfigDict` change (only the `model_` *prefix* is protected, and `model_config` specifically is impossible — it is the Pydantic config attribute, already used at `serialize.py:53`). The original `agent_model` was over-cautious. Bundled into this session's single `WIRE_SCHEMA_VERSION` 2 → 3 break (one contract break, not two; same `AgentSpec` under redesign, user-requested — not a smuggled orthogonal rename).
- **Resolution = a dedicated closed registry, NOT `__all__`, NOT reflection.** A module-level `AGENT_KIND_REGISTRY: Dict[str, Type[Agent]]` co-located with the subclasses is **the single source** — the exact `det_nodes.NODE_KIND_BEHAVIOUR` analog (`det_nodes.py:188-209`). Each specialized subclass declares its own stable wire key as a class attribute (`WIRE_KIND: ClassVar[str]` — the `RESERVED_NAME` analog, `det_nodes.py:79,111,139`). The reverse `class → kind` map is **derived by inverting the registry**, never hand-maintained twice (`RESERVED_NAME_TO_KIND` pattern, `det_nodes.py:198-200`). A CI test cloned from `tests/coordination/topology/test_node_kind_registry.py` asserts registry membership and that every `WIRE_KIND`-bearing class is registered and vice-versa — a forgotten agent fails the build, not silently at a user's load. `marsys.agents.__all__` is the package public API (includes `BaseAgent`/`AgentPool`/`MemoryManager`/registry — `__init__.py:32-64`) and is **not** the source (validator-refuted §2 of the draft). Adding a specialized agent tomorrow = one `WIRE_KIND` attr + one registry entry, no dispatch edits.
- **`params` = typed per-subclass `*ParamsSpec`, curated, no secrets, no runtime objects.** `WebSearchAgentParamsSpec`, `CodeExecutionAgentParamsSpec`, … each a hand-curated declarative subset — exactly how base `AgentSpec` is a curated subset of `Agent.__init__` and `ModelConfigSpec` is a curated mirror that drops `api_key` (`models/serialize.py:50-75`). Non-JSON / runtime-derived kwargs (`filesystem`, `code_config`, `root_dir = base_directory or Path.cwd()` — `code_execution_agent.py:78`) are **not on the wire**; the subclass `__init__` reconstructs them from declarative inputs on hydrate exactly as it does today (pass `base_directory` or omit → its default fires). "Every wire object is a typed mirror" is the established wire principle.
- **Behaviour in the class.** Only identity + declarative `params` travel. The subclass `__init__`/factory rebuilds its own tools/instructions/methods on hydrate — the `NODE_KIND_BEHAVIOUR` / name-only-tools+`tool_registry` / `handler_registry` precedent. No subclass behaviour is serialized.

## 3. Resolved design decisions (were §3 forks — decided with the user 2026-05-17; ADR-009-gated)

1. **Async construction → B′: a single always-async hydrate entrypoint.** `pydantic_to_agents` and `pydantic_to_topology` become `async`. Rationale (first-principles, user-decided after deep tradeoff): hydration is *intrinsically* async (a `WorkflowDefinition` may contain a `BrowserAgent`, whose browser only initializes in `await create_safe()` — `browser_agent.py:1126,1166`); the honest type of the operation is `async`. A dual sync/async API (rejected option A) forces the caller to choose based on workflow contents they have not parsed yet — structurally impossible to do correctly. A faked sync wrapper (rejected option B) hides `asyncio.run` and raises `RuntimeError: asyncio.run() cannot be called from a running event loop` in the dominant deployment (already inside a loop, hydrate-then-`await Orchestra.run`); the only "fixes" (`nest_asyncio` global monkeypatch imposed on consumers; thread-offload that blocks the consumer loop through a real browser launch) are worse. Deferring construction into `Orchestra.run` (rejected option C) breaks S08's fully-materialized-`Topology` contract and touches the TRUNK-CRITICAL orchestrator. Internally the async entrypoint dispatches via `AGENT_KIND_REGISTRY` using the framework's own established idiom `if hasattr(cls, 'create_safe'): await cls.create_safe(**params) else: cls(**params)` (`pool_factory.py:44-49`, `agent_pool.py:156,209`). **Cost (accepted):** migrate the in-repo sync callers — `examples/session08_workflow_roundtrip.py:156` + ~6 test callsites (`test_serialize.py:86,113,151,317,331`, `test_s08_coverage_gaps.py:85`) — to `await`/`asyncio.run`. Spren mirrors with its own models and does not import these functions; its alignment is the separately-contracted follow-up regardless, so B′ adds no Spren cost. v0.3 is pre-release — set the honest contract before consumers multiply.
2. **`params` shape → typed per-subclass `*ParamsSpec`** (not one open `dict`). The codebase convention is "every wire object is a typed mirror" (`ModelConfigSpec`, base `AgentSpec` — both curated typed subsets). An open dict would special-case secrets/`Path` per subclass at validation anyway (same work, less safety, breaks the convention, defers the break to a user's load).
3. **Secrets → dropped from the wire by construction.** No credential kwarg appears in any `*ParamsSpec` (the `ModelConfigSpec`-drops-`api_key` precedent, `models/serialize.py:50`). The subclass `__init__` already resolves missing keys from env vars (`web_search_agent.py:50-53`: `google_api_key=None` → `GOOGLE_SEARCH_API_KEY`); that *is* the resolver, unchanged. Optionally an injected secrets map may be threaded at the async hydrate entrypoint, mirroring `tool_registry`/`handler_registry` injection (`agents/serialize.py:130`, `topology/serialize.py:474`). This pattern never references the Session-07 `runnable` flag, so it is **stable across Session 10** (`10-runtime-credential-resolution.md` / ADR-010, which later collapses the dual `ModelConfig` path — out of scope here).
4. **`agent_model` → `model`** — yes, bundled (see §2; verified clean).
5. **Wire-contract + Spren coordination.** `WIRE_SCHEMA_VERSION` 2 → 3 (additive-discriminator change class, same as S08's 1→2 — `topology/serialize.py:81`); `x-wire-schema-version` re-embeds (`:588`). Spren-mirror update is **surfaced, not implemented** (anti-pattern #8; Spren = contracted follow-up). CHANGELOG + DEPRECATIONS note the bump.

## 7. Risks / premise seeds (for the Phase-A→B Premise Ledger)

- **Additive-base-surface regression (load-bearing).** If a subclass is routed through a `params`-only path, every base field silently regresses. Make "base surface still round-trips for a subclass" an explicit acceptance criterion + characterization regression pin (byte-identical base-`Agent` default; per-subclass full round-trip equality including base fields).
- **`params` non-JSON/secret/runtime leakage.** Each `*ParamsSpec` must be characterized from the subclass's real `__init__` (primary source) — verify no `RunFileSystem`/`CodeExecutionConfig`/`ModelConfig`/credential reaches the wire; test that a serialized spec contains no secret and no machine-specific path.
- **Async migration blast radius.** Every in-repo sync caller must move; a missed one fails at runtime. Grep-confirm the full caller set in B1 (do not trust §3.1's count); full framework suite must be green vs the S08-merged baseline.
- **BrowserAgent async path.** Round-trip must reconstruct a *browser-ready* BrowserAgent via the registry `create_safe` dispatch — not a sync-constructed, browser-unready instance. Test the actual async hydrate.
- **Wire-contract change with a live mirror consumer (Spren).** Same discipline as S08: schema version bump, no dual-field shim, surfaced not silent; Spren update is the contracted follow-up.
- **`kind` vs ADR-008 `agent_type`.** ADR-009 must explicitly record that `kind` supersedes ADR-008 Decision 9's literal `agent_type` wording (a recorded user decision — mirrors `NodeSpec.kind`), or a future reader hits a contradiction.

## 8. Test strategy (outline — `acceptance.md` is authoritative once frozen)

Unit: registry single-source + derived reverse map + CI registration test (cloned shape); each `*ParamsSpec` curation (no secret/runtime fields). Integration: per-subclass `agent → AgentSpec → JSON → AgentSpec → agent` round-trip asserting subclass identity + base surface + declarative params; `kind="agent"` byte-identical base round-trip (characterization pin); secrets-off-wire assertion; async BrowserAgent hydrate produces a browser-ready instance. Regression: full framework suite vs S08-merged baseline (zero new failures); migrated example runs. Live: only where a touched path has analogous existing live coverage (the S08 round-trip example pattern) — surface if new live infra would be needed rather than inventing it.
