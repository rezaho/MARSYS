# v0.3 — MVP Detailed Plan

The first MARSYS umbrella release that includes Spren. Framework continues at its own version line (v0.2.x as of planning); the umbrella release that introduces Spren bumps the umbrella version to v0.3.

## Outcome

A single user installs Spren via the native installer (or any secondary channel — brew / winget / apt / npm / pipx / Docker), opens the app, and uses a continuously-active personal AI assistant (the meta-agent) to:

1. **Build workflows visually** — 3-agent multi-agent topologies in an `@xyflow/react` canvas with model picker, agent forms, pattern presets, and inline lint feedback
2. **Import workflows from Python code** — drop a `.py` file using the marsys framework; Spren parses agents + topology + configs and materializes a workflow record (provenance: `code_import`)
3. **Run workflows** — manually triggered from the UI OR via the meta-agent ("run workflow X with this input"), watching live AG-UI events stream in
4. **Inspect runs** — nested span trace viewer with token, latency, and cost chips per span; full run history filterable by status/time/workflow
5. **Talk to the meta-agent** on the home page — NOT a chat-primary surface — a four-surface command center: Now / Since-you-were-away / Activity / Chat input
6. **See what the agent is doing** — live activity stream of agent thoughts and actions; clear "since you were away" inbox for items needing decision
7. **Edit memory in `vim`** — the markdown KB at `<data-dir>/sandbox/shared/memory/` is human-editable; the agent picks up edits on next index pass
8. **Trust the cost ceiling** — per-day budget cap enforced at the runtime level; agent pauses non-critical work when over budget
9. **Trust hard rails** — destructive actions ALWAYS require confirmation regardless of standing approvals

All running locally, with the user's own LLM API keys stored in OS keychain (or encrypted SQLite fallback in headless modes).

## Hard constraints (apply to every session)

1. **No mocked feature implementations in product code** (SP-007). Test fixtures at external boundaries are fine.
2. **No backward-compat code paths** (SP-006). Migrations for stored data are allowed.
3. **TRUNK-CRITICAL framework files untouched** (SP-001). v0.3 expects two framework PRs that land via the framework's own contribution flow, NOT as Spren-branch edits: the NDJSON streaming tracing writer (framework Session 01, already shipped) and the AG-UI event stream translator (framework Session 06, lands before Spren Session 04). Both are framework-internal — Spren consumes; Spren does not author framework code.
4. **Bounded triggers** (SP-011). Meta-agent wakes on inbox events, heartbeats, sub-instance completions, and agent-scheduled events. NO ad-hoc polling loops finer than the heartbeat.
5. **Hard rails always confirm** (SP-012).
6. **Cost ceiling is load-bearing** (SP-013).
7. **Working instances persist scratchpad atomically** (SP-014). N/A in v0.3 (no sub-instances) but the pattern is established for v0.4.
8. **Untrusted-channel writes never live-touch memory** (SP-015).
9. **Markdown is source of truth for memory** (SP-016).
10. **Forget tombstones, never deletes** (SP-017).
11. **Framework knows nothing of Spren** (SP-018). Three doors only: `Orchestra.run()`, `EventBus.subscribe()`, `TelemetrySink`. The framework also ships an optional `AGUIEventStream` adapter (Session 06) built on top of `EventBus` — Spren consumes it but does not modify it. The framework knows about the open AG-UI standard, not about Spren.
12. **API is the single source of truth** (SP-019). Every client renders the same FastAPI surface; no UI logic in backend; no hand-written client types mirroring Pydantic.
13. **Every session ships unit + integration + E2E tests** that apply to its work.
14. **Every session produces a working artifact** — never "halfway through, will finish next."

## Risks specific to v0.3

| Risk | Mitigation |
|---|---|
| **Framework PR (AG-UI translator, Session 06) regresses framework users** | Translator lives in a new `marsys.transport.aggui` (or equivalent) namespace; no existing framework module touched. Framework regression suite must pass with zero diff outside that namespace. Land via framework's own PR flow with reviewers. |
| **AG-UI protocol still evolving** | Framework Session 06 pins to a specific AG-UI version; framework ships `schema_version` on every event for forward compatibility; use AG-UI's `Custom` event for marsys-specific events that don't fit; revisit on AG-UI stabilization. |
| **Framework Session 06 delays block Spren Session 04** | If the AG-UI framework PR isn't merged + released before Spren Session 04 wants to start, Spren Session 04 ships against a draft branch of the framework PR (vendor in via uv branch reference) and switches to the published version on merge. Do NOT fork a Spren-side translator as a "temporary" workaround — that becomes permanent debt. |
| **PyInstaller per-platform builds are slow** | Aggressive caching in CI; parallel matrix; revisit if pipeline blows past 30 min. |
| **Apple notarization adds release latency** | Notarization runs async (minutes-to-hour); release flagged "in notarization" until done; CI does not block on it for non-mac artifacts. |
| **Code-signing certificate procurement** | Apple Developer Program + Windows EV cert procurement happens BEFORE Session 10 begins. Block release ship until done. |
| **Cost rate table goes stale** | YAML file; flag for periodic update; don't crash if a model is missing — warn and emit cost=0 for that span. |
| **OS keychain UX (macOS first-prompt)** | In-app explainer before triggering the prompt; user knows what they're approving. |
| **Meta-agent quality bar is high** (central UI) | v0.3 ships read-only + low-risk-writes ONLY. Hard rails on destructive. Buggy meta-agent can annoy but cannot destroy. |
| **Memory poisoning attacks** (MINJA / MemoryGraft class) | SP-015: queue-then-consolidate; no live-write from any non-user-direct source. v0.3 has no remote channels yet so attack surface is minimal, but the defense is in place from day one. |
| **Sandbox permission wrapper bugs** | Application-layer wrapper has comprehensive unit tests + a `verify-permissions` CLI; defense in depth via Docker mode for production-ish use. |
| **Daemon crashes** | Durable inbox in `<data-dir>/sandbox/inbox/pending/`; on restart, replay; SP-014 atomic scratchpad (moot until v0.4 sub-instances). |
| **Tauri sidecar PID is opaque** (PyInstaller wraps) | Stdin shutdown command, not `process.kill`. Build into sidecar protocol from Session 01. |
| **Auto-updater swap fails mid-install** | Atomic rename pattern; rollback to prior version on failed verify. Tested in CI. |
| **Scope creep into v0.4 territory** | Hard rule: if a session needs sub-instances, teams, vector recall, TUI, or Tauri-polish features, it's a v0.4 session. Don't bolt them on. |
| **Session size growing** | Each session is sized for one focused work-block. If a session balloons mid-implementation, the implementer must ASK before silently expanding scope. |

## Rollback strategy

- **Per-session:** revert the PR. Sessions are scoped to be reversible.
- **Framework AG-UI translator (Session 06):** if the AG-UI adapter surfaces unforeseen incompatibilities with `EventBus`, revert the framework PR, ship v0.3 with raw `EventBus.subscribe()` exposed to Spren's SSE endpoint (Spren translates inline as a documented temporary), and schedule the framework adapter for v0.3.1.
- **Native installer build catastrophe on a single platform:** ship v0.3 from the platforms that work; document the gap; release v0.3.1 with the missing platform. Don't block all-platform release on one platform's signing or notarization issue.
- **Meta-agent runtime crash on first launch (catastrophic):** ship a fallback "minimal mode" CLI flag (`spren launch --no-meta-agent`) that boots Spren without starting the daemon — user can still use visual builder + run inspection.
