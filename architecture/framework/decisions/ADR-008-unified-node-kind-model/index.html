    ADR-008: Unified Node-Kind Model for the Canonical Topology & Wire Format¶

    Status: Proposed (awaiting written approval — escalation gate) Date: 2026-05-16 (drafted); 2026-05-16 (revised after Phase-A validation refuted the v1 mechanism — see §Revision note) Implements: Framework Session 08 — docs/implementation/framework/sessions/v0.3.0/08-unified-node-kind-model.md Related: ADR-005 (unified-barrier algorithm — the runtime that consumes Start/End/User), ADR-006 (deprecation timeline — the removals whose destination model this builds), Session 04 (the serializer this corrects), Session 07 (exercised this path; did not touch the node model)

    Revision note (why v2)¶

    The v1 draft asserted the orchestrator core stays untouched because "TopologyGraph det-node queries resolve via Node.kind". Phase-A validation refuted that from primary source: TopologyGraph holds NodeInfo (graph.py:38-48, field node_type, no kind, no core.Node); its get_start_node/is_det_node/get_det_node resolve from a separate det_nodes: Dict[str, DeterministicNode-instance] registry (graph.py:1042-1075) populated by register_det_node, fed by analyzer._add_nodes' isinstance check and the shim — there is no path from core.Node.kind to that registry. The diagnosis (serializer is backwards; unify at the core) survived; the mechanism and blast radius did not. v2 relocates the real work to the analyzer/parsing materialization seam and states the true (larger) surface.

    Context¶

    coordination/topology/serialize.py is the canonical cross-process wire shape. It exposes Literal["user","agent","system","tool"] (serialize.py:87) mirroring NodeType (core.py:20-25), and workflow_to_pydantic raises NonSerializableTopologyError on any DeterministicNode (serialize.py:279-286) — a frozen Session-04 criterion (AC-59) on the rationale "det-nodes carry execution-runtime state beyond what NodeSpec captures" (exceptions.py:15-25).

    That rationale is false from primary source: StartNode/EndNode.__init__ carry only name (det_nodes.py:77-78,109-110); UserNode's only non-trivial field is handler (det_nodes.py:137-139) — a runtime binding identical to agent tool callables, already handled by name + tool_registry at hydrate (serialize.py:31-34). The wire never needed det-node state — only the discriminator, which is a string, exactly what node_type already is.

    The deeper defect: the v0.3 redesign already made explicit Start/End/User det-nodes "the canonical way to express workflow entry, termination, and user-channel I/O" (DEPRECATIONS.md:24-31). The runtime Topology accepts them (core.py:167-174); the unified-barrier orchestrator drives from them (orchestrator.py:162-168). The Session-04 serializer mirrored the legacy enum-only model and banned the canonical one — built backwards relative to the framework's documented direction. The canonical wire path runs today only because the REMOVE-IN-V0.4 shim (orchestra.py:_apply_legacy_topology_shim, ~456-616) synthesises the missing Start/End/User. At v0.4 the crutch is deleted and the format cannot express a runnable workflow; a UserNode workflow already cannot round-trip.

    Compounding structural problems (all primary-source confirmed): two muddled axes (executor-kind class hierarchy vs NodeType enum, with User in both — core.User(Node) at core.py:132 and det_nodes.UserNode — forcing the parsing.py:67 carve-out); vestigial NodeType.SYSTEM/TOOL (never dispatched); specialized Agent subclasses don't round-trip (agents/serialize.py:158 always rebuilds base Agent; AgentSpec has no type discriminator).

    The three representations (the corrected mental model)¶

    kind must thread through three layers, not two:

    1. Wire — WorkflowDefinition/NodeSpec (Pydantic, JSON). Discriminator field.
    2. Canonical runtime model — core.Topology/core.Node (the "canonical internal representation", core.py:1-7). What pydantic_to_topology produces and Orchestra.run receives.
    3. Execution graph — TopologyGraph/NodeInfo + the det_nodes instance registry (graph.py:38-48,1042-1075). Built by TopologyAnalyzer (analyzer._add_nodes) and the shim, from (2). Consumed by the Orchestrator only via the TopologyLike Protocol (orchestrator_types.py:198-205, orchestrator.py:163).

    The real work is the materialization seam between (2) and (3): a core.Node with kind ∈ {START,END,USER} (or a "Start"/"End"/"User" string via parse_node) must be turned into the right DeterministicNode instance via one authoritative registry and register_det_node'd. That single seam replaces the analyzer's isinstance branch (analyzer.py:185-192), the parsing.py:62-72 string special-casing, and closes the round-trip gap that a deserialized core.Node(kind=START) does not currently become a det-node.

    Why the orchestrator core genuinely does not change (narrow, true claim)¶

    get_start_node/is_det_node/get_det_node keep reading TopologyGraph.det_nodes; the TopologyLike signature and the Orchestrator dispatch body are unmodified. We change what feeds det_nodes (kind-materialization in the analyzer instead of isinstance/string), not what reads it. init_workflow precedence is confirmed (orchestrator.py:154-168): entry_agent set → direct spawn; else → require get_start_node(). The Orchestrator's TRUNK-CRITICAL body is untouched — but the analyzer/parsing/graph materialization path is in scope and is the real work, not a one-line branch swap. This is why this is implementer-tier with an ADR gate, not an Orchestrator rewrite.

    Why "exactly one Start" can be enforced now¶

    [Corrected 2026-05-17 — Step-0 characterization refuted the v2 wording of P6; primary-source-traced true mechanism below.]

    get_start_node() already raises TopologyError on multiple Starts (graph.py:1069-1074). The earlier claim that a pure-agent topology has no Start (and runs via the _find_entry_agents heuristic) is false. True mechanism, traced + empirically confirmed: TopologyAnalyzer.analyze unconditionally writes graph.metadata["exit_points"] = <terminal nodes> (analyzer.py:119-127, comment "ALWAYS find and store exit points"); the shim's exit_points block synthesises an EndNode + edges; det_nodes is now non-empty, which un-gates the shim's final block (orchestra.py:599) → it synthesises exactly one Start. So for every topology with ≥1 terminal node the shim already produces exactly one Start + End today; the _find_entry_agents heuristic is already dead for them. The only surviving zero-Start case is a degenerate topology with no terminal node (a pure cycle where every node has an outgoing edge, or an isolated single node): exit_points empty → no End → det_nodes empty → final block skipped → no Start → today the silent heuristic guesses or dies vaguely. The enforcement work therefore reduces to one additive post-shim guard: after _apply_legacy_topology_shim, if get_start_node() is None, raise an explicit TopologyError naming the degenerate-no-terminal issue (before the legacy _find_entry_agents fallback, which is left in place for v0.4). Multiple-Start already enforced; single-Start already synthesised for all non-degenerate topologies. Net guarantee delivered: every runnable topology provably has exactly one Start or fails loudly. Behaviour change is confined to the degenerate no-terminal input (already a malformed/no-exit workflow) — flagged + documented.

Frame holds: this finishes the v0.3 redesign in the component that missed it.

    Decision¶

    Y-statement. In the context of the canonical topology + wire format, facing a serializer that mirrors the legacy enum-only node model and bans the canonical det-nodes on a refuted premise, we decided to make a single closed NodeKind discriminator the canonical node taxonomy (owned in topology/core.py, mirrored 1:1 on the wire), with deterministic behaviour materialized from kind at the analyzer._add_nodes seam (parse_node normalizes to uniform core.Node; Topology.nodes is homogeneous) via one authoritative extension-open registry — and neither a parallel public class hierarchy nor a vestigial enum — to achieve a canonical format that losslessly round-trips every runnable workflow and survives the v0.4 shim removal, accepting a wire-schema value-set change (one in-repo mirror consumer, contracted follow-up), a cross-package RESERVED_NODE_NAMES + agent-registration behaviour change, and a one-session blast radius spanning core + serialize + analyzer/parsing + pattern_converter + the equality oracle + characterization tests.

    1. topology/core.py — NodeType → NodeKind = {AGENT, START, END, USER}; drop SYSTEM/TOOL (P5); remove User(Node) (core.py:132); Node carries kind; Topology holds uniform Nodes (no dual-family accept at core.py:171). RESERVED_NODE_NAMES is derived from NodeKind (frozenset(k.name.lower() for k in NodeKind if k is not NodeKind.AGENT)), NOT a rewritten literal — it stays a module-level frozenset with the same name and import path (a downstream consumer imports the value; derive, don't restructure the export). Surfaced behaviour change (P10): agents may now be named system/tool; may no longer be named Start/End; the Hypothesis strategy (strategies.py:22) and any downstream import are affected.
    2. Behaviour registry (single-sourced) — a NodeKind → behaviour-class map is the single source of truth; the reserved-name string lookup ("Start"→StartNode) is derived from the classes' existing RESERVED_NAME attrs (det_nodes.py:75,107,135), NOT kept as a second hand-maintained RESERVED_DETNODE_NAMES dict. RESERVED_DETNODE_NAMES, the parse_node carve-out (parsing.py:67), and RESERVED_NODE_NAMES (core.py:29) are three hand-synced spellings of one fact — collapse to one (AC-4 "extension-open" is only true if adding a kind touches exactly one place). DeterministicNode/StartNode/EndNode/UserNode classes stay as behaviour. No speculative conditional/loop classes built (anti-pattern #11).
    3. Materialization seam (the real work) — Option A, true uniform Topology.nodes. parsing.parse_node returns a uniform core.Node(kind=...) for every input — a "Start"/"End"/"User" string → core.Node(name, kind=START/END/USER); it no longer returns DeterministicNode instances (reversing today's parsing.py:56-69 behaviour). Topology.__post_init__ accepts Node only (the (Node, DeterministicNode) dual-accept at core.py:171 is removed). The DeterministicNode behaviour instance is materialized only at analyzer._add_nodes: it branches on node.kind, builds the instance via the registry, and calls register_det_node — retained, graph.py:1042 keeps its sole caller; the isinstance at analyzer.py:185 becomes a kind check at the same site (not deleted — that v1-reflex wording would strand register_det_node). Every reader doing isinstance(x, DeterministicNode) on Topology.nodes/a parsed node migrates to node.kind — bounded set: parsing.py:56, the serialize.py:279 check being deleted anyway, core.py:171, analyzer.py:185. (Option B — keep instances in Topology.nodes, just add kind — is the bounded-depth Fix #2 already rejected; it leaves the dual-type model alive and would make Decision 1's "uniform Nodes" false. Chosen: A.) Single chokepoint confirmed by primary source: every runnable topology flows _ensure_topology→analyzer.analyze→_add_nodes (orchestra.py:936-955, analyze called once @948); pydantic_to_topology builds a graph-free core.Topology (serialize.py:454) — no bypass. Closes P4b (a deserialized core.Node(kind=START) now materializes as a det-node at _add_nodes).
    4. topology/serialize.py — NodeSpec.node_type → kind (1:1 NodeKind); workflow_to_pydantic total; delete NonSerializableTopologyError (exceptions.py); pydantic_to_topology gets a handler_registry param (tool-registry pattern) for UserNode; update topology_equals oracle: node_type.value → kind.value AND add is_convergence_point to node_key (P12). The wire serializes + round-trips is_convergence_point (serialize.py:119,305,447) but the oracle (serialize.py:225-226) is blind to it — "full round-trip equality" is unsatisfiable for an is_convergence_point=True node otherwise. Any convergence round-trip discrepancy the oracle exposes — whether in S08's own changed path OR pre-existing and unrelated — is in-scope to fix in S08 (user-directed 2026-05-17: the serializer must be correct when S08 lands). Recorded as a deliberate, user-instructed scope inclusion (the anti-pattern #8 "no smuggled scope" exception is explicit user instruction) so a future reader sees it was intentional, not creep. Tradeoff accepted: larger diff; rollback couples the node-kind change with the convergence fix. Decided at acceptance-freeze — NOT deferred to sign-off.
    5. pattern_converter.py (7 sites, P11) — Node(name="User", node_type=NodeType.USER) → kind=USER. Pattern semantics/edges unchanged; only the node constructor migrates. NOT a "pattern model change".
    6. coordination/orchestra.py — NO production code (final, 2026-05-17, user-approved "clean option"). Characterization proved "exactly one Start" is already fully enforced for the explicit/canonical path: find_entry_point_with_manual hard-errors on no-entry/cycle (graph.py:516), find_exit_points_with_manual (strict) hard-errors on no-terminal (graph.py:644), surviving topologies get exactly one synthesized Start, multiple-Start errors (graph.py:1069). A post-shim guard would be defensive code for an impossible state (anti-pattern #5). Step 6 = regression-pins only: test_s08_characterization.py f1/f2 lock that the existing analyze-layer TopologyErrors remain. The _find_entry_agents heuristic is reachable only via auto_run soft-exit mode — a separate subsystem, out of S08 scope.
    7. Wire backward-compat — wire validator stays permissive in v0.3: legacy-no-Start deserialises + DeprecationWarning; stored kind/value system/tool → reject at load with a migration message. Hard invariant lives at the runtime/post-shim seam. v0.4: warning→error, shim deleted.
    8. AC-59 reversal is an explicit, intentional spec change — test_serialize.py:612-649 asserts pytest.raises(NonSerializableTopologyError); Decision 4 reverses it. Those tests are inverted to assert successful round-trip, surfaced in the session sign-off and the S04 acceptance amendment (anti-pattern #1: never a silent test deletion).
    9. Specialized-agent gap — principle here, implementation Session 09. The unifying principle ("a typed graph entity round-trips via a closed wire discriminator + a framework-side class registry; behaviour in the class; bindings injected at hydrate") is the canonical pattern for both axes. AgentSpec agent_type discriminator + agent-class registry = ADR-blessed Session 09 (P7), deliberately not bundled — scope discipline.

    Scope boundaries¶

    • In S08: Decisions 1-8. End-to-end node-kind unification incl. the analyzer materialization seam, the total-shim + complete Start invariant, the equality-oracle update, pattern_converter node migration, the RESERVED_NODE_NAMES reconciliation with its behaviour/cross-package change surfaced, characterization tests written first (no AC-11 baseline exists), the AC-59 inversion, DEPRECATIONS/CHANGELOG/schema-version, S04 acceptance amendment.
    • Out of S08: agent_type round-trip (Session 09); v0.4 shim/heuristic deletion; downstream hand-mirror drop + stored-data migration (a consumer-side PR, P9); edge/pattern semantics; new det-node classes.
    • Test bar: AC-15 green = the framework suite. A downstream consumer's import does NOT break (derived frozenset, same name/path); its suite diverges only where it asserts the old reserved-value semantics or relies on a stale local NodeType (still has system/tool) / mirror — the contracted P9 follow-up. If a downstream suite runs in the same bar, Step 0 scopes the green target explicitly; the divergence is expected/surfaced, not "fixed" by reverting the framework change.

    Consequences¶

    • Positive: canonical format losslessly round-trips every runnable workflow incl. explicit entry/exit/human nodes; survives v0.4 shim removal with zero further model change; one single-sourced taxonomy; two-axes muddle + User double-representation gone; Start invariant complete now; future det-kinds additive.
    • Negative: wire discriminator name+value-set change → schema bump; a downstream hand-mirror + its suite diverge until a follow-up PR (P9, contracted); RESERVED_NODE_NAMES membership change alters agent-registration legality (surfaced); undeterminable-entry topologies now error instead of guessing (intended).
    • Risks: round-trip regressions on exotic combos → exhaustive NodeKind×EdgeType×pattern matrix + Hypothesis; legacy/parity outcomes changing under the node-kind migration → characterization-test-first (5 cases incl. degenerate) then parity assert; the analyzer materialization being mis-scoped → P4b is a named load-bearing premise the validator re-confirms; RESERVED_NODE_NAMES cross-package breakage → surfaced as expected P9 fallout, green-bar scoped in Step 0; oracle change hiding a real round-trip bug → oracle update is itself reviewed; the degenerate-no-terminal guard is a behaviour change for an already-malformed input → flagged + documented (CHANGELOG/DEPRECATIONS), characterized pre/post. The corrected-P6 episode itself shows characterization-first working as intended (false premise caught before any production code).

    Alternatives Rejected¶

    1. Bounded-depth (wire+dispatch only; keep core duality + User dup) — smaller, but ships a wire more coherent than its runtime and leaves the root mess as debt. Rejected: user chose full unification; mirroring the mess isn't "canonical".
    2. Honesty-only (drop "runnable" claim, keep the reject) — does not fix v0.4 incompatibility or the authoring gap. Rejected.
    3. Two-field kind+subtype — premature; family is derivable from kind + registry. Rejected (anti-pattern #4/#11); revisit if a kind needs sub-variants.
    4. Delete _find_entry_agents/entry_agent now — already v0.4-scheduled; smuggled scope, touches the run signature. Rejected.
    5. Coerce stored system/tool → agent on load — silently rescues a never-runnable doc. Rejected: reject-at-load with migration message (serializer's fail-fast-at-storage value, serialize.py:161-169).
    6. Bundle agent_type into S08 — same mechanism, doubles scope beyond the reported problem. Rejected; ADR-blessed Session 09 so the principle isn't silently half-applied.
    7. Split core-model and wire-mirror into two sessions — ships a half-migrated dual model (the exact legacy-retention anti-pattern). Rejected: one coherent change, one session.

    Resolved open decisions¶

    • Wire field rename node_type→kind (value-set changes regardless; one mirror consumer; honest name).
    • Reserved names: one set = non-agent kinds, lowercase, case-insensitive; system/tool removed; behaviour + cross-package change explicitly surfaced (P10), not silent.
    • Vestigial-value load policy: reject (alt. 5).
    • Specialized-agent scope: principle here, implementation Session 09 (Decision 9 + alt. 6).
    • is_convergence_point in the equality oracle: include it in node_key (the wire round-trips it; an oracle blind to it cannot verify "full round-trip equality"). Decided here, at acceptance-freeze. Any convergence round-trip discrepancy the oracle exposes (in-path OR pre-existing/unrelated) = fix in S08 (user-directed 2026-05-17; deliberate scope inclusion, not creep; tradeoff: coupled rollback + larger diff).

    Migration & backward-compat¶

    Legacy no-Start stored WorkflowDefinition: still deserialises, DeprecationWarning, runs via the now-total shim — unchanged user-visible outcome (parity characterization-pinned). Legacy entry_point/exit_points/User(Node): shim path preserved + parity-pinned. node_type(old) vs kind(new): schema bump + CHANGELOG; framework ships new shape, a downstream consumer migrates its store + mirror in its own PR (P9). No dual-field wire shim (anti-pattern: legacy retention) — the version boundary is the contract.
